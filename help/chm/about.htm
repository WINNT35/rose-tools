About Rose Tools

Background

Rose Tools began as an effort to bring VMware guest services to the ReactOS community. Porting individual tools without a shared core proved unstable, so the decision was made to port open-vm-tools as a whole; providing a solid foundation that future components can build upon. The goal is to make ReactOS significantly more usable when running inside VMware virtual machines.

What Rose Tools Does

Version 0.1 is a proof of concept focused on establishing a working core rather than end-user features. It includes:

  • vmrosd.exe — the Rose Tools daemon; loads plugins and manages the RPC channel between the guest and VMware host.
  • checkvm.exe — detects whether the system is running inside a VMware virtual machine and reports the tools version.
  • Heartbeat — signals to the VMware host that guest tools are active and running.
  • GuestInfo — periodically collects and reports guest system information to the VMware host, including network configuration, OS details, disk usage, and system statistics.

While modest on the surface, v0.1 establishes the plugin architecture and core protocol layer that all future components will depend on. Currently tested on Windows XP (NT 5.1). See Installation for compatibility details.

What Rose Tools Does Not Do Yet

Features such as drag and drop, clipboard synchronization, and automatic display resizing are planned but not yet implemented. These will be introduced in future releases as plugins built on top of the v0.1 core.

Relationship to open-vm-tools

Rose Tools is derived from open-vm-tools. The guest protocol layer is preserved as closely as possible, modified only where necessary for Windows NT compatibility. The core carries no dependency on GLib. While the protocol and core libraries are largely derived from upstream, the outer ring (the services and executables) is written from scratch for the Win32 API.
