Contributing

Rose Tools welcomes contributions from the community. This page provides a brief overview of the project structure, coding standards, and licensing requirements for contributors.

Build Requirements

  • MinGW (GCC for Windows)
  • C89 compatible compiler
  • Windows XP or ReactOS (for testing)

Coding Standards

The preferred standard for Rose Tools is C89, particularly for the outer ring. This ensures compatibility with older toolchains and keeps the codebase consistent with the project's targets. Contributions should follow C89 where practical:

  • Prefer /* */ block comments over //
  • Declare variables at the top of a block where possible
  • No GLib or other Unix-specific dependencies
  • Win32 API only for platform-specific code

Project Structure

Rose Tools is primarily divided into two distinct layers:

  • lib/ Ś the core; contains the guest protocol implementation and shared libraries. This layer is heavily derived from open-vm-tools and should be modified as little as possible. Changes here should only be made where strictly necessary for Windows NT compatibility.
  • services/ Ś the outer ring; contains the daemon, utilities, and plugins. This is where the majority of new development takes place. New features should be implemented as plugins under services/plugins/ wherever possible.

When in doubt, ask yourself whether your code belongs to the protocol and core infrastructure (lib/) or to the services and tools built on top of it (services/). Keep the two layers cleanly separated.

Writing a Plugin

New features should be implemented as plugins loaded by vmrosd.exe rather than standalone executables or modifications to the daemon itself. This keeps vmrosd.exe stable and makes features independently maintainable. Refer to the existing guestInfo plugin as a reference implementation.

Licensing

All contributions must use the correct license for their layer:

  • lib/ Ś GNU Lesser General Public License v2.1 only
  • services/ Ś GNU General Public License v2.0 or later

Every source file must include the appropriate license header. See existing files for the correct header format. If a layer is not mentioned please refer to the "COPYING" file inside that folder.

Getting Involved

Bug reports, feature requests, and pull requests are welcome on the GitHub repository. When submitting a bug report please include your VMware version, hardware compatibility mode, and the exact output or error message you received.
