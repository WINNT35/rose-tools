Frequently Asked Questions

Do I as an end user have any obligations?

For most users, no. If you are simply running or sharing Rose Tools unmodified, you have no obligations to worry about. If you modify the source code and distribute your changes, standard GPL terms apply; but this only affects developers who are actively changing and redistributing the software, not regular users. Rose Tools is free and open source software provided at no charge. If you paid for this software, it was likely redistributed without added value.

Can I use the core library in my own project?

Yes. The LGPL v2.1 permits you to use the library in your own project, including proprietary projects, provided that any modifications you make to the library itself are released under the LGPL v2.1. You are not required to license your entire project under the LGPL.

Does the LGPL core affect the license of the outer ring?

No. The LGPL v2.1 only requires that modifications to the library itself remain under the LGPL. Code that uses the library, such as the services and executables in Rose Tools, is free to use a different license. This is the intended use case of the LGPL and why it exists as a distinct license from the GPL.

What are my obligations if I fork Rose Tools?

If you fork Rose Tools and distribute it, the GPL requires that you make the complete source code publicly available under the same terms and you must preserve all existing license notices. Each component may carry a different license; always check the license header of the individual file or the COPYING file in the relevant directory. You may not relicense any component without the consent of the copyright holders. However, the combined work as a whole may be distributed under a later version of the GPL, such as GPL v3.0.

The full license texts are available in the GitHub repository. Each component directory contains a COPYING or LICENSE file with the relevant license text. The root LICENSE file covers the project as a whole.
