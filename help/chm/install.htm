Installation

Rose Tools v0.1 requires no installation. Both executables can be run directly from the command prompt.

Running vmrosd.exe

Launch vmrosd.exe from the command prompt. If Rose Tools is functioning correctly, you should see the following message:

pluginMgr: plugin 'guestInfo' loaded.

Additionally, the primary IP address of your virtual machine should appear in VMware's console view.

Running checkvm.exe

Launch checkvm.exe from the command prompt. If Rose Tools is functioning correctly inside a VMware virtual machine, you should see:

VMware software version 6 (good)

If you are running checkvm.exe outside of a VMware virtual machine, the following message is expected and indicates correct behavior:

Not running inside a VMware virtual machine.

You can also run checkvm.exe with the -p flag to print the product name of the detected VMware platform:

checkvm.exe -p

This will print one of the following; VMware Workstation, VMware ESX Server, or Unknown.

Troubleshooting

If either executable does not produce the expected output, please submit an issue on the GitHub repository. Include the following details in your report:

  • VMware version
  • Whether you are running a lower hardware compatibility mode
  • The exact output or error message you received

Compatibility

Rose Tools v0.1 has been tested on Windows XP Media Center Edition 2005 SP3 (NT 5.1) 32-bit under VMware Workstation. Support for ReactOS, Windows NT 4.0, Windows NT 3.51, and 64-bit guests is planned for a future release.

Note: a graphical installer is planned for a future release.
